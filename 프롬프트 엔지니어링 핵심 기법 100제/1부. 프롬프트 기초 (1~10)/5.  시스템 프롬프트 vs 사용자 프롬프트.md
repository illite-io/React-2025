# 5. 시스템 프롬프트 vs 사용자 프롬프트

## 시스템 프롬프트(System Prompt)

* 모델이 기본적으로 따르는 **최상위 지시문**
* 사용자가 직접 수정하기 어려운 경우가 많음
* 모델의 **행동 방식, 톤, 역할** 등을 정의
* 예:

  ```text
  너는 친절한 AI 어시스턴트다. 항상 정중하게 대답하고 예시를 들어 설명한다.
  ```

---

## 사용자 프롬프트(User Prompt)

* 사용자가 입력하는 **일반적인 질문이나 요청**
* 매번 대화할 때마다 추가되는 입력
* 시스템 프롬프트에 영향을 받지만, 구체적이면 우선 적용됨
* 예:

  ```text
  프랑스 혁명을 초등학생 눈높이에 맞게 설명해줘.
  ```

---

## 차이점 정리

| 구분       | 시스템 프롬프트       | 사용자 프롬프트      |
| -------- | -------------- | ------------- |
| 역할       | 모델의 기본 성격 정의   | 사용자의 직접 요청    |
| 변경 가능 여부 | 제한적 (플랫폼에서 설정) | 자유롭게 작성 가능    |
| 지속성      | 대화 내내 유지       | 입력마다 달라짐      |
| 예시       | "정중하게 답해라"     | "간단히 표로 정리해줘" |

---

## 결론

* **시스템 프롬프트**는 모델의 뼈대를 잡고,
* **사용자 프롬프트**는 상황별 세부 지시를 내린다.
